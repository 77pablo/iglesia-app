# Reporte: Reportes y estadísticas del pastor

## Backend
- Nuevo router `backend/src/reportes.js`, montado en `/api/reportes` (`backend/src/server.js`).
- Todo el módulo exige `esPastor` (403 para cualquier otro rol, incluido obispo/tesorero/líder/feligrés).
- `GET /api/reportes/asistencia` → `{ mensual:[{mes,total}], porGrupo:[{mes,grupo_id,grupo,total}] }` (agregación con `strftime('%Y-%m', evento.fecha)`).
- `GET /api/reportes/tesoreria` → `{ mensual:[{mes,ingresos,gastos,saldo}], saldoTotal }`.
- `GET /api/reportes/crecimiento` → `{ mensual:[{mes,altas}], totalActivos }` (altas por `persona.creada_en`; activos = `activo=1`).
- `GET /api/reportes/export.csv?tipo=asistencia|tesoreria|crecimiento` → CSV con BOM UTF-8 (igual patrón que `panel.js`), `tipo` inválido → 400. Cada export queda auditado (`auditar(...,'exportar_reporte',...)`).
- `backend/src/auth.js`: se agregó el módulo `'reportes'` a `modulosVisibles` para pastor y super_admin.

## Frontend (`web/app.js` + `web/styles.css`)
- Nueva entrada de menú **"Reportes"** (ícono 📈), visible solo si el rol tiene el módulo `reportes` (mismo mecanismo que el resto del NAV).
- `vistaReportes()`: tarjetas de KPI (asistencia último mes, saldo total, miembros activos, altas del mes) + 3 gráficos en `<canvas>` dibujados a mano (línea para asistencia, barras agrupadas ingresos/gastos, barras para altas) — sin librerías externas (CSP no permite CDNs). Funciones reutilizables `trazarLineas` / `trazarBarras` / `_prepararCanvas`.
- Manejo de estado vacío por sección ("Aún no hay datos de..." / "...movimientos de tesorería").
- Botones: **Exportar CSV** (uno por tipo, reutiliza el patrón blob/descarga de `panel.js`) y **Imprimir** (`window.print()`).
- CSS: `.chart-legend` para leyenda de colores y un bloque `@media print` (oculta sidebar/topbar/botones vía clase `.no-print`, deja solo el contenido).
- Todo el texto dinámico usa `escHtml`/interpolación de números controlados por el backend (sin inyección de HTML de usuario).

## Pruebas
- `backend/test/reportes.test.js` (7 tests, `node:test`, reutiliza `helpers.js`): agregación correcta de asistencia por mes/grupo, ingresos/gastos/saldo mensual, altas por mes + total activos (incluye caso de persona inactiva), acceso exclusivo del pastor (feligrés y líder de grupo → 403), aislamiento estricto por `iglesia_id`, y export CSV (BOM, encabezados, tipo inválido → 400).
- `node --test`: **60/60 verdes** (suite completa del backend, sin regresiones).

## Preocupaciones / notas
- Los endpoints son exclusivos del pastor (no del obispo), siguiendo la instrucción explícita del encargo; si se desea que el obispo también observe reportes por iglesia, habría que sumar `esObispo` como en `panel.js`.
- Los canvases se dibujan una sola vez al renderizar la vista (tamaño según `clientWidth` en ese momento); no hay listener de `resize` — aceptable para el alcance, pero si se cambia el tamaño de ventana con la vista abierta el gráfico no se reajusta hasta volver a entrar a la vista.
- `npm install` fue necesario en `backend/` (no había `node_modules`) para poder correr los tests.
